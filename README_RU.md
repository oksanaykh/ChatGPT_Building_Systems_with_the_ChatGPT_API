### 1. Введение в архитектуру LLM
Современные LLM обучаются методом прогнозирования следующего токена. Важно различать типы моделей:

*   **Base LLM:** Обучена на массивах данных для продолжения текста.
*   **Instruction Tuned LLM:** Оптимизирована через SFT и RLHF для следования инструкциям.

#### Токенизация
*   1 токен ≈ 4 символа (для английского языка).
*   Лимиты контекстного окна (например, 4000 токенов для GPT-3.5) включают и запрос, и ответ.

### 2. Безопасность и чат-формат
Для управления диалогом используются три роли:
*   `System`: Глобальные инструкции и ограничения.
*   `User`: Запрос пользователя.
*   `Assistant`: История ответов модели.

> **Важно:** Никогда не хардкодьте API-ключи. Используйте `.env` файлы и библиотеку `python-dotenv`.

### 3. Оценка и классификация входов
*   **Классификация:** Разделение запросов на категории (например, `billing`, `tech_support`) через JSON для маршрутизации.
*   **Moderation API:** Проверка контента на насилие, ненависть и другие нарушения.
*   **Защита от инъекций:** Использование разделителей (например, `####`) и проверочных промптов для выявления попыток обхода инструкций.

### 4. Методы обработки
#### Chain of Thought (Цепочка рассуждений)
Модель генерирует промежуточные шаги перед ответом. Для чистоты интерфейса используется «Внутренний монолог», который скрывается от пользователя программно.

#### Chaining Prompts (Цепочка промптов)
Разбиение сложной задачи на модули.
*   **Плюсы:** Снижение стоимости, упрощение тестирования, возможность интеграции внешних API между шагами.

### 5. Верификация выходов
Перед выдачей ответа пользователю рекомендуется:
1. Провести повторную модерацию.
2. Выполнить **Fact-Checking**: сравнить ответ с эталонным контекстом для исключения галлюцинаций.

### 6. Стратегии оценки (Evaluation)
Разработка итеративна: от 1-5 примеров до автоматизированных тестов.
*   **Оценка по рубрикам:** Использование GPT-4 как судьи.
*   **OpenAI Evals:** Сравнение с эталоном (Subset, Superset, Discrepancy).

### 7. Комплексный пример: Support Assistant
Типовой Pipeline системы:
1. `Входная модерация` → 2. `Классификация` → 3. `Retrieval (Поиск данных)` → 4. `Генерация` → 5. `Выходная модерация`.

### 💡 Лучшие практики
*   Воспринимайте LLM как **Reasoning Agent** (агент для рассуждений), а не как статичную базу знаний.
*   Модульность (цепочки коротких промптов) всегда надежнее одного универсального промпта.
